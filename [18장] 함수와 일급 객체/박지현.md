## 함수와 일급객체
### 일급 객체
  - 무명의 리터럴로 생성될 수 있다(런타임에 생성가능하다)
  - 변수나 자료구조에 저장할 수 있다.
  - 함수의 매개변수에 전달할 수 있다.
  - 함수의 반환값으로 사용할 수 있다.
- 이 조건을 만족하는 객체를 일급객체라고 한다.
- 함수는 위의 모든 조건을 만족하므로 일급 객체다.
- 객체는 값이므로 함수는 값과 동일하다.
- 일급 객체로 함수가 가장 큰 특징은 반환 값으로 함수를 사용할 수 있다.
- 이것은 함수형 프로그래밍을 가능하게 한다.
- 함수는 객체인데 일반 객체는 호출할수 없지만 함수는 호출할 수 있다.

### 함수 객체의 프로퍼티

- 함수를 getOwnPropertydescriptor로 보면 함수 객체의 데이터 프로퍼티(arguments, caller, length, name, prototype)와 __proto__가 있다.
- Object.prototype객체의 프로퍼티를 상속받은 것이다.

#### arguments 프로퍼티
- 함수 호출이 전달된 인수들의 정보를 담고 있는 유사 배열 객체
- 함수의 매개변수와 인수의 개수가 일치하는지 확인하지 않아서 개수가 안맞아도 에러가 발생하지 않는다.
- 적게 전달하면 인수에 undefined가 들어가고 초과하여 전달하면 인수에 다 넣고 남은 것은 arguments에 남아있다.

> Symbol.iterator 프로퍼티
> 이 프로퍼티를 통해 이터레이터로 순회할 수 있지만
> 매개변수 이름에 의미가 있을 텐데 매개변수를 이렇게 쓰는 것은 뭔가 이상하다.
> 가변 인자함수를 구현할 때는 유용할 것 같다. 예를 들어 sum
> 근데 rest라는 파라미터가 있으니까 애매할 것 같기도 하다.
> rest는 ES6에만 있어서 알아놓으면 좋을 듯

#### caller 프로퍼티
- ECMAScript 사양에 포함되지 않은 비표준 프로퍼티
- 자신을 호출한 함수를 가르킨다.

#### length 프로퍼티
- 함수를 정의할 때 선언한 매개변수의 개수
- argument 프로퍼티는 인자의 개수, 합수 객체의 length는 매개변수의 개수

#### name 프로퍼티
- name 프로퍼티는 함수의 이름

#### __proto__접근자 프로퍼티
- 모든 객체는 [[Prototype]] 이라는 내부 슬롯을 가진다.
- [[Prototype]]은 객체지향 프로그래밍의 상속을 구현하는 프로토타입 객체다.
- __proto__를 통해 간접적으로 접근할 수 있다. 내부 슬롯은 직접 접근할 수 없다.

##### prototype 프로퍼티
- 함수가 객체를 생서앟는 생성자 함수로 호출될 때 생성자 함수가 생성할 인스턴스의 프로토타입 객체를 가르킨다.
